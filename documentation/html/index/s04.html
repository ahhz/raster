supported datatypes for raster creation

There are several functions to create gdal_rasters. These provide a raster class associated with geotiff raster dataset that are optimized for the moving window library. It is recommended but not strictly necessary to have a one-to-one match between the value_type of the raster class and the GDALDataType of the raster dataset. A condition is that the value_type specified with template argument T must fit within the number of bytes available.

When the datatype is not specified by the user it must be one of the types in the table below, it is derived from the value_type of the raster, using native_value_typeT::type.

When the datatype is specified by the user it must be one of the types in the table below. A condition is that the value_type specified with template argument T must fit within the number of bytes available.

Table 1.1. Supported datatypes

T default datatype (GDALDataType) type
unsigned char GDT_Byte 8 bit unsigned integer
unsigned short GDT_UInt16 16 bit unsigned integer
short GDT_Int16 16 bit signed integer
unsigned int GDT_UInt32 32 bit unsigned integer
int GDT_Int32 32 bit signed integer
float GDT_Float32 32 bit floating point
double GDT_Float64 64 bit floating point
